# New Project Checklist

## Purpose

Provide a repeatable checklist for setting up a new software project before writing code.

A clean project setup reduces confusion, improves organization, and creates a strong foundation for development.

---

## When to Use

Use this checklist at the start of every new project, including:

- Client websites
- Web applications
- School projects
- Portfolio projects
- Practice builds

---

## Project Setup

- [ ] Define the project name
- [ ] Define the project goal
- [ ] Identify the target users
- [ ] Identify the main features
- [ ] Choose the technology stack
- [ ] Create the project folder
- [ ] Create the GitHub repository
- [ ] Clone the repository locally
- [ ] Open the project in VS Code

---

## Repository Setup

- [ ] Add `README.md`
- [ ] Add `.gitignore`
- [ ] Add initial folder structure
- [ ] Confirm Git is tracking the project
- [ ] Make the first commit
- [ ] Push to GitHub

---

## Development Setup

- [ ] Install dependencies
- [ ] Configure environment variables
- [ ] Add example environment file if needed
- [ ] Confirm the application runs locally
- [ ] Confirm no sensitive data is committed

---

## Documentation Setup

- [ ] Add project description
- [ ] Add setup instructions
- [ ] Add tech stack
- [ ] Add project status
- [ ] Add planned features
- [ ] Add known issues if applicable

---

## First Commit Message

Recommended:

```bash
git commit -m "Initial project setup"
```
