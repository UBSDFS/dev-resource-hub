# Development Workflow

## Purpose

Document the standard development process followed throughout the software development lifecycle.

A structured workflow reduces mistakes, improves consistency, and ensures every feature is planned, developed, tested, and documented before completion.

---

## When to Use

Follow this workflow whenever implementing a new feature, fixing a bug, or making significant changes to an application.

---

# Software Development Lifecycle

## 1. Understand

- Review project requirements
- Understand the business problem
- Identify the desired outcome
- Clarify assumptions before coding

---

## 2. Plan

- Break the work into smaller tasks
- Identify dependencies
- Determine the approach
- Estimate effort

---

## 3. Design

- Sketch ideas if needed
- Review UI/UX requirements
- Consider architecture
- Plan data flow
- Review API requirements

---

## 4. Develop

- Implement one feature at a time
- Write clean, readable code
- Keep commits small and meaningful
- Follow project conventions

---

## 5. Test

- Verify expected functionality
- Test edge cases
- Test responsive layouts (if applicable)
- Ensure no existing functionality is broken

---

## 6. Review

Before committing, ask:

- Does it work?
- Is the code readable?
- Can it be simplified?
- Did I introduce unnecessary complexity?
- Is documentation needed?

---

## 7. Commit

- Stage changes
- Write a meaningful commit message
- Commit only related changes

---

## 8. Push

- Push to GitHub
- Verify remote repository
- Confirm project builds successfully

---

## 9. Reflect

After completing the task:

- What did I learn?
- What could be improved?
- Should anything be added to the Dev Resource Hub?

---

## Guiding Principle

Software development is an iterative process.

Understand the problem.
Plan the solution.
Build with intention.
Test thoroughly.
Document what you learn.
