# Git & GitHub Workflow

## Purpose

Establish a consistent Git workflow that promotes clean version control, meaningful project history, and reliable collaboration.

---

## Philosophy

Commit early.
Commit often.
Commit with purpose.

Each commit should represent a meaningful, testable unit of work that moves the project forward.

Git is more than a backup system—it documents the evolution of the project.

---

## Commit Standards

- Make small, focused commits.
- Complete and test a feature or logical change before committing.
- Use clear, descriptive commit messages.
- Avoid mixing unrelated changes in a single commit.
- Push changes regularly to keep the remote repository up to date.

---

## Commit Message Format

Use an action verb.

Examples:

- Add user authentication
- Create project setup checklist
- Update dashboard layout
- Refactor API routes
- Fix login validation bug
- Document deployment workflow

Avoid generic messages such as:

- Updates
- Fixed stuff
- Changes
- Misc

---

## Daily Workflow

1. Pull the latest changes.
2. Create or update a feature.
3. Test functionality.
4. Review changes.
5. Stage changes.
6. Commit with a meaningful message.
7. Push to GitHub.

---

## Guiding Principle

Version control should tell the story of the project.

Every commit should clearly communicate what changed and why.
